How State Categories Affect Seat Blocking

State categories determine how seats are grouped into blocks in the List View. Seats are grouped together when they share the same:

  • Event - Must be in the same event
  • Ticket Type - Must have the same ticket type
  • Section - Must be in the same section
  • Row - Must be in the same row
  • Direct Hold - Both must have the same direct hold (or both have none)
  • Direct Kill - Both must have the same direct kill (or both have none)
  • Availability - Both must be "For Sale" or both "Not For Sale"
  • State Category - Both seats' states must belong to the same state category (as configured below)

Note: Seats must also be consecutive (adjacent seat numbers) to be grouped into a block. Each block displays a summary row showing the seat range, and can be expanded to show individual seat details.

To configure which states belong to which category, drag state pills between category cards. The first category (by display order) that contains a seat's state will be used for blocking.
